# 6.1 更新日志

相比 6.0 版本，6.1 主要围绕项目维护、构建一致性和回归检查做了改进。

## 6.1.6 修复（未发布，提交于 main）

- 焦点在滑块、下拉框、按钮等控件上时，方向键 / 空格 / R / F 不再被全局快捷键劫持，控件恢复原生键盘行为。
- `.txt` 文件导入在 UTF-8 解码失败时回退 GB18030，Windows 记事本默认保存的 ANSI/GBK 稿件不再乱码。
- 垂直镜像模式下编辑稿件不再跳回顶部，任何镜像状态下编辑都保持当前进度；顺带修复暂停状态下拖动进度后播放锚点过期的问题。
- 编辑抽屉打开时点击进度条不再误关抽屉。
- 修复 Electron 和 Tauri 两端重开第二屏时缓存状态与实时状态乱序的问题，第二屏不再从错误的进度或暂停状态开始；Tauri 侧不再向隐藏的第二屏推送状态。
- 新增上述场景的 Playwright 回归测试。

## 工程整理

- Tauri 版和 Electron 版统一使用 `6.1.0` 产品版本号。
- 新增 `VERSION` 统一版本文件，并完善版本同步与检查脚本。
- Tauri 主窗口和第二屏不再维护两份重复页面，第二屏改为通过 `index.html?mode=output` 进入输出模式。
- Tauri 和 Electron 的前端入口页面保持一致，减少两边功能漂移。

## 自动化检查

- 新增 `npm run check` 项目级检查命令。
- 新增前端一致性检查，确保 Tauri / Electron 入口页面内容一致。
- 新增 Playwright 冒烟测试，覆盖主界面渲染、稿件编辑显示和第二屏输出模式。
- 测试优先使用系统 Chrome / Edge，避免额外下载浏览器导致检查卡住。

## 文档和发布

- 更新 README、打包说明、发布清单和维护说明。
- 发布前检查流程改为统一运行版本检查、前端同步检查和基础 UI 测试。
