# 6.2 更新日志

6.2 版本包含一批影响日常使用的 bug 修复，以及底部控制栏的重新设计。

## 6.2.1

- 第二屏输出按钮和显示器选择器移到播放操作旁边，不再放在控制栏最右侧。
- 速度滑块加宽并弹性填充第一行中间区域，消除大片空白。

## 控制栏改版

- 控制栏从三栏网格改为两行布局：第一行是录制高频操作（播放、回到顶部/尾部、速度、第二屏输出、稿件、读取），第二行是设置项（镜像、显示模式、字号/行距/字距、宽度/水平位置/指示线）。
- 默认高度自动适配内容，完整展开两行，不再出现默认被裁切、内部出滚动条的情况；手动拖过的高度仍然记住。
- 移除控制栏内重复的横向进度条（预览窗右侧已有垂直进度条和百分比）。
- 镜像提示改为设置行内联显示，快捷键提示挪到控制栏右上角。

## 问题修复

- 焦点在滑块、下拉框、按钮等控件上时，方向键 / 空格 / R / F 不再被全局快捷键劫持，控件恢复原生键盘行为。
- 播放中按 ↑↓ 或滚轮调整进度不再被动画循环拉回原位置；鼠标点开始播放后方向键立即可用。
- `.txt` 文件导入在 UTF-8 解码失败时回退 GB18030，Windows 记事本默认保存的 ANSI/GBK 稿件不再乱码。
- 垂直镜像模式下编辑稿件不再跳回顶部，任何镜像状态下编辑都保持当前进度；顺带修复暂停时拖动进度后播放锚点过期的问题。
- 编辑抽屉打开时点击进度条不再误关抽屉。
- 修复 Electron 和 Tauri 两端重开第二屏时缓存状态与实时状态乱序的问题，第二屏不再从错误的进度或暂停状态开始；Tauri 侧不再向隐藏的第二屏推送状态。

## 工程

- GitHub Actions 升级到基于 Node.js 24 的版本（checkout/setup-node/upload-artifact v5），消除弃用警告。
- 新增覆盖上述场景的 Playwright 回归测试，测试总数达到 17 个。
