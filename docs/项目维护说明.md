# 项目维护说明

本文档用于约束项目的源码目录、版本号和发布文件，避免开发副本、历史构建产物和发布包混在一起。

## 唯一正式源码目录

当前正式源码目录为仓库根目录，也就是 `提词器-整理版-6.0`。

后续开发、提交、推送 GitHub、打包发布，都应以这个目录为准。`new-chat` 根目录里的旧版本文件、历史安装包、旧 `.app`、旧 `.zip` 和临时实验目录只作为历史资料，不再作为正式源码来源。

## 目录职责

```text
.
├── windows-tauri/       Tauri 桌面版源码
├── windows-electron/    Electron 桌面版源码
├── scripts/             版本同步和检查脚本
├── docs/                打包、发布、更新日志和维护说明
├── VERSION              当前统一产品版本号
├── README.md            项目总说明
└── 使用指南.md          用户使用说明
```

## 不提交的内容

以下内容不应进入仓库：

- `node_modules/`
- `dist/`
- `target/`
- 安装包和便携包，例如 `.exe`、`.msi`、`.zip`
- 临时日志和系统文件

这些规则已经写入 `.gitignore`。发布包应上传到 GitHub Release，不放进源码仓库。

## 版本号规则

Tauri 版和 Electron 版使用统一产品版本号。当前版本写在 `VERSION`。

修改版本号时先运行：

```powershell
.\scripts\sync-version.ps1 -Version 6.1.0
```

发布前检查：

```powershell
.\scripts\sync-frontend.ps1
.\scripts\check-version.ps1
.\scripts\check-frontend-parity.ps1
```

完整检查可以运行：

```powershell
npm install
npm run check
```

检查通过后再打包。

版本脚本会同步 Tauri / Electron 的 `package.json`、`package-lock.json`、Tauri `Cargo.toml`、Tauri 配置和前端页面里的短版本号。

## 后续结构优化方向

当前 Tauri 和 Electron 的入口页面内容保持一致：

- Tauri 主窗口加载 `windows-tauri/web/index.html`
- Tauri 第二屏通过 `index.html?mode=output` 进入输出模式
- Electron 加载 `windows-electron/index.html`

修改前端界面或提词逻辑时，优先修改 `windows-tauri/web/index.html`，然后运行 `.\scripts\sync-frontend.ps1` 同步到 Electron。发布前运行 `.\scripts\check-frontend-parity.ps1` 检查。

下一阶段可以继续把这份共享前端页面拆到公共源码目录，再由两个桌面壳各自加载同一套构建结果。
