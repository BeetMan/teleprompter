# 项目维护说明

本文档用于约束项目目录、统一版本号和发布流程，避免开发副本、历史构建产物和正式发布文件混在一起。

## 正式源码目录

当前正式源码目录为仓库根目录，也就是 `提词器-整理版-6.0`。后续开发、提交 GitHub 和本地打包都应以这个目录为准。

## 目录职责

```text
.
├── windows-tauri/       Tauri 桌面版源码
├── windows-electron/    Electron 桌面版源码
├── scripts/             版本同步和检查脚本
├── tests/               Playwright 自动化测试
├── docs/                打包、发布、更新日志和维护说明
├── VERSION              当前统一产品版本号
├── README.md            项目总说明
└── 使用指南.md          用户使用说明
```

## 不提交的内容

以下内容不应进入源码仓库：

- `node_modules/`
- `dist/`
- `target/`
- 安装包和便携包，例如 `.exe`、`.msi`、`.zip`
- 临时日志、测试截图和系统文件

这些规则已经写入 `.gitignore`。正式发布包由 GitHub Actions 上传到 GitHub Release，不放进源码仓库。

## 版本号规则

Tauri 版和 Electron 版使用统一产品版本号。当前版本写在 `VERSION`，现在为 `6.4.0`。

修改版本号时运行：

```powershell
.\scripts\sync-version.ps1 -Version 6.4.0
```

版本同步后运行完整检查：

```powershell
npm install
npm run check
```

脚本会同步 Tauri / Electron 的 `package.json`、`package-lock.json`、Tauri `Cargo.toml`、Tauri 配置和前端页面里的短版本号。Tauri 构建时如果更新了 `Cargo.lock`，也应一并提交。

## 前端同步规则

当前 Tauri 和 Electron 的入口页面内容保持一致：

- Tauri 主窗口加载 `windows-tauri/web/index.html`
- Tauri 第二屏通过 `index.html?mode=output` 进入输出模式
- Electron 加载 `windows-electron/index.html`

修改前端界面或提词逻辑时，优先修改 `windows-tauri/web/index.html`，然后运行：

```powershell
.\scripts\sync-frontend.ps1
.\scripts\check-frontend-parity.ps1
```

## 自动发布

工作流位于 `.github/workflows/release.yml`。推送符合 `vX.Y.Z` 格式的标签后，GitHub Actions 会：

1. 安装 Node.js 和 Rust 环境
2. 执行 `npm run check`
3. 构建 Electron 便携版和安装版
4. 构建 Tauri 安装版
5. 生成 `SHA256SUMS.txt`
6. 创建对应的 GitHub Release 并上传安装包

示例：

```powershell
git tag -a v6.4.0 -m "Release v6.4.0"
git push origin v6.4.0
```

当前工作流默认生成未签名安装包。接入 Windows 代码签名证书后，应将证书配置放入 GitHub Actions Secrets，并在构建步骤中增加签名流程，不要把证书文件或密码提交到仓库。
